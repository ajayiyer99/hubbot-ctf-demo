About this demo

  
HubBot Catch & Cage is the interactive front end for the Innovation Hub prompt-injection → Sentinel → auto-remediation demo.

  
    
    
  • Theater (CTF): a visitor tries to jailbreak HubBot — extract the planted flag, leak its system prompt, or abuse the lookup_vip honeypot.
    • 
    
  • Benign prompts are logged as Informational Azure OpenAI audit entries (no alert). Attack attempts are evaluated by Azure AI Content Safety Prompt Shields / Defender for Cloud threat protection for AI and raise a Microsoft Defender for Cloud alert at its catalog severity (Jailbreak & Credential-Theft = Medium, ASCII Smuggling = High) in Microsoft Sentinel (SecurityAlert).
    • 
    
  • The correlated Sentinel incident (SecurityIncident) is prioritized High and, ~15s after the prompt, runs the block playbook — HubBot's Entra identity is disabled (accountEnabled=false) and its sessions revoked. Shown here as a simulated timeline.
    • 
    
  • Lobby: ambient attract loop for the lobby TV — sanitized incident replay, no live chat.
    • 
  

  
All VIP data is synthetic. The block API asserted is the verified Entra Agent ID disable path; "Agent 365" is not claimed. See webapp/README.md.
